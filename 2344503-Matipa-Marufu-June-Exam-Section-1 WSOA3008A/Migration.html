“Digital technologies rely on hardware that has short life expectancy” (Rubio and Wharton, 2020). Keeping a digital artwork on an aging piece of hardware, leaves that data at risk. The software and the environment ages therefore that artwork is more likely to become inaccessible. For example, if the machine is damaged, and the artwork on that machine is not backed up online, the artwork will be lost. The same applies if the artwork is of a certain format which is no longer readable by current software. When technology is updated old technology is replaced. If the artwork is not constantly moved to environments supported by the product’s creators, it will reach a stage where the updated software will be unable to access the old file even if the original hardware is still perfectly intact. The same applies for artworks composed via code, as the compiler can only translate certain high level languages into binary for the machine to read. (Wieringa, 2017)
